\begin{abstract}
This report presents my solution for Data Mining Assignment 3 on Human Activity Recognition using wrist accelerometer data. Each sample is a 5-minute sequence with 300 ordered time steps and six raw accelerometer statistics. The task is to classify each complete sequence into one of six activity labels using Macro F1-score. My final approach treats each CSV file as one sequence-level sample, applies strict file-level label alignment, constructs motion-based temporal features, extracts ROCKET-style random convolution features, and trains a balanced RidgeClassifierCV model. The best observed Kaggle public leaderboard score during experimentation was \textbf{0.7752}. The final reproducible submitted pipeline achieved \textbf{0.7746}. This report includes preliminary analysis, preprocessing analysis, temporal alignment, ablation study, leaderboard evidence, failed-experiment analysis, and reproducibility instructions.
\end{abstract}
\section{Project Summary}
The goal of this assignment is to classify human activities from wrist-mounted accelerometer readings. The dataset consists of 5-minute windows, where each CSV file contains 300 rows corresponding to one-second intervals. Each row contains six raw features: \texttt{mean\_x}, \texttt{mean\_y}, \texttt{mean\_z}, \texttt{std\_x}, \texttt{std\_y}, and \texttt{std\_z}.
The most important modeling decision is that the task is not row-level classification. Instead, each CSV file corresponds to one complete 5-minute activity window and has exactly one activity label. Therefore, the task was formulated as sequence-level classification:
\[
X_i \in \mathbb{R}^{300 \times 6}, \quad y_i \in \{0,1,2,3,4,5\}.
\]
The final method combines chronological sorting by \texttt{index}, validation of \texttt{file\_id}, magnitude-based channels, energy-based channels, temporal derivative features, segment-based statistics, ROCKET-style random convolution features, and balanced RidgeClassifierCV classification.
The development process was iterative. Early classical feature-engineered models reached around 0.75--0.76 on the public leaderboard. The best observed Ridge-based temporal feature model reached \textbf{0.7752}. The final reproducible implementation reached \textbf{0.7746}. Additional variants, including larger ensembles, rare-class boosting, and prediction smoothing, were tested but did not consistently improve generalization. Therefore, the final submitted implementation prioritizes reproducibility, stability, and correct sequence-level processing.

\subsection{Raw Data Observations}
The raw data is already aggregated into one-second intervals. Therefore, classical statistical features are meaningful, but the temporal evolution across the 300-second window remains important.
Important observations from the raw data were:
\begin{itemize}[noitemsep]
    \item Mean-axis features alone are not sufficiently discriminative because different activities can have overlapping average acceleration.
    \item Standard-deviation channels better describe activity intensity and motion variability.
    \item Magnitude features reduce sensitivity to wrist orientation.
    \item Temporal differences help distinguish activities with similar global statistics but different movement dynamics.
    \item User variability makes user-aware validation important.
\end{itemize}
These observations directly informed the method design: the model uses derived motion channels, temporal segment features, convolutional ROCKET features, and class-balanced learning.
\section{Proposed Method}
\subsection{File-Level Label Alignment}
The label alignment is performed at the CSV-file level. For each file, all 300 rows are sorted by \texttt{index}, and the full ordered sequence is treated as one training sample:
\[
X_i = [x_{i,0}, x_{i,1}, \ldots, x_{i,299}], \quad y_i \in \{0,1,2,3,4,5\}.
\]
The model never treats the 300 rows as independent labeled examples. This prevents label leakage and preserves the correct task formulation. Each prediction in the final submission corresponds to exactly one CSV file.
\subsection{Data Validation}
Before feature extraction, the implementation checks that:
\begin{itemize}[noitemsep]
    \item each CSV file contains exactly 300 rows,
    \item all required feature columns exist,
    \item \texttt{file\_id} is unique within each file,
    \item the filename matches the \texttt{file\_id},
    \item each training file contains exactly one activity label,
    \item all labels are valid integers in the range 0--5.
\end{itemize}
These checks ensure that the preprocessing and Kaggle submission remain aligned with the assignment requirements.
\subsection{Derived Motion Channels}
The original six channels are extended with physically motivated features. The acceleration magnitude is computed as:
\[
|\mathbf{a}| = \sqrt{x^2+y^2+z^2}.
\]
The standard-deviation magnitude is computed as:
\[
|\boldsymbol{\sigma}| = \sqrt{\sigma_x^2+\sigma_y^2+\sigma_z^2}.
\]
First-order temporal differences are computed as:
\[
\Delta x_t = x_t - x_{t-1}.
\]
The magnitude transformation reduces sensitivity to wrist orientation and captures the overall movement intensity independently of the coordinate system. Difference features capture second-to-second changes in motion and are useful for distinguishing activities with similar global acceleration distributions but different dynamic patterns.
The final derived channels include magnitude, standard-deviation magnitude, first-order axis differences, magnitude difference, and energy:
\[
E_t = x_t^2 + y_t^2 + z_t^2.
\]
Non-finite values are replaced with zero after feature construction.
\subsection{Normalization}
A \texttt{StandardScaler} is fitted only on the training sequences. The data is reshaped from \((N,300,C)\) to \((300N,C)\), scaled per channel, and reshaped back. The same scaler is then applied to the test set. This prevents information leakage from the test data and stabilizes the high-dimensional RidgeClassifierCV training process.
\subsection{Statistical and Segment Features}
For each channel, global statistics are extracted, including mean, standard deviation, minimum, maximum, median, percentiles, variance, skewness, kurtosis, and RMS. Segment-level features are also computed by splitting the 300-second window into temporal segments.
The segment features encode how motion evolves over the five-minute interval. This is important because two activities may have similar average acceleration but different temporal progression.
\subsection{ROCKET-Style Temporal Features}
ROCKET-style random convolution features are used to capture local temporal patterns. Each random kernel is applied to the sequence, and three values are extracted:
\[
f_1 = \max_t (K*x)_t,
\]
\[
f_2 = \frac{1}{T}\sum_t \mathbb{1}[(K*x)_t > 0],
\]
\[
f_3 = \frac{1}{T}\sum_t \max(0,(K*x)_t).
\]
These represent maximum activation, proportion of positive values, and mean positive activation. ROCKET features are useful because they capture short- and medium-range temporal patterns without requiring a deep neural network.
The final submitted implementation uses 5,000 ROCKET kernels. This setting was selected as a practical balance between runtime and performance.
\subsection{Final Classifier}
The final model is a RidgeClassifierCV with balanced class weights:
\[
w_c = \frac{N}{K \cdot N_c}.
\]
Here, \(N\) is the number of training samples, \(K\) is the number of classes, and \(N_c\) is the number of training samples in class \(c\). This weighting compensates for class imbalance and improves Macro F1-score by increasing the influence of minority-class samples.
\subsection{Why the Final Model Does Not Use the Full Ensemble}
During development, multiple ensemble strategies were explored extensively, including combinations of RidgeClassifierCV, Logistic Regression, ExtraTrees, and hard-voting aggregation schemes. The original motivation behind these ensembles was to increase robustness by combining models with different inductive biases and decision boundaries.
However, empirical experiments demonstrated that increasing ensemble complexity did not consistently improve generalization performance on the Kaggle leaderboard. In particular, the strongest single Ridge-based temporal model repeatedly outperformed larger voting systems.

The experiments revealed several important findings:
\begin{itemize}[noitemsep]
    \item RidgeClassifierCV generalized more consistently across validation folds and public leaderboard evaluations.
    \item ExtraTrees-based predictions introduced higher variance and unstable rare-class predictions.
    \item Hard-voting ensembles occasionally suppressed correct minority-class predictions generated by the Ridge model.
    \item Additional post-processing strategies, including prediction smoothing and aggressive rare-class boosting, frequently reduced the Macro F1-score.
    \item Larger ensemble systems significantly increased implementation complexity while providing limited or inconsistent leaderboard improvements.
\end{itemize}
These observations suggest that the gains from larger ensembles were not stable enough to justify their increased complexity and reduced reproducibility. Therefore, the final implementation intentionally prioritizes reproducibility, stability, deterministic behavior, correct sequence-level processing, and robust temporal feature extraction over ensemble complexity.

\subsection{Analysis of Failed Experiments}
Several attempted improvements did not improve leaderboard performance:
\begin{itemize}[noitemsep]
    \item Hard voting over weaker models sometimes reduced the score compared to RidgeClassifierCV alone.
    \item Aggressive rare-class boosting increased false positives and reduced Macro F1.
    \item Prediction smoothing across neighboring files harmed minority-class detection.
    \item Large automatic experiment generators created many submissions but did not improve the best public score.
\end{itemize}
These failed experiments were important because they showed that the best final solution should remain simple, reproducible, and focused on the strongest observed model family instead of maximizing ensemble complexity.
\section{Discussion}
The experiments demonstrate that Human Activity Recognition from wrist accelerometer data benefits strongly from sequence-aware temporal modeling. While classical global statistics already provide substantial predictive power, additional temporal features improve discrimination between activities with similar average motion intensity.
ROCKET-based random convolution features proved useful because they capture local temporal patterns without requiring computationally expensive deep learning models. The combination of statistical and convolution-based representations resulted in a robust and computationally feasible pipeline.
Another important observation is that increasing model complexity does not necessarily improve generalization performance. Several ensemble variants produced unstable leaderboard behavior. In contrast, the Ridge-based pipeline generalized more consistently and produced the strongest balance between stability, reproducibility, and leaderboard performance.
This final decision is especially important in the context of the assignment, because the private test data may be more difficult than the public leaderboard split. A reproducible and stable pipeline is therefore preferable to a highly complex approach that only performs well on one public submission.

\section{Conclusion}
This assignment required careful treatment of accelerometer data as sequence-level samples. The final method aligns labels at the file level, preserves chronological order, validates \texttt{file\_id}, constructs motion-based temporal features, applies ROCKET-style random convolution features, and trains a balanced RidgeClassifierCV.
The best observed public leaderboard score during experimentation was \textbf{0.7752}. The final reproducible submitted pipeline achieved \textbf{0.7746}. More complex ensembles and post-processing did not consistently improve performance. The final cleaned solution therefore produces exactly one \texttt{submission\_final.csv} file using the strongest reproducible Ridge-based pipeline.
The most important findings are:
\begin{itemize}[noitemsep]
    \item The HAR task must be treated as sequence-level classification.
    \item Temporal information is important for distinguishing activities.
    \item Macro F1-score requires careful handling of minority classes.
    \item ROCKET-derived features improve temporal representation.
    \item Simpler Ridge-based classification generalized more stably than larger ensembles.
\end{itemize}
